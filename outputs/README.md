# Outputs 運用ガイド

このフォルダは、特定のプロジェクト（記事作成、開発など）の成果物と、公開済みのアーカイブを管理する場所です。

## 階層構造
- **`projects/`**: 進行中のプロジェクト単位のフォルダ。
    - `[project_name]/`: 記事ごとの専用空間（research, drafts, assets を内包）。
- **`published/`**: 公開済み、または最終決定されたコンテンツの保管庫。
- **`archive/`**: 古いバージョンや、完了したプロジェクトの全ての資料の避難所。

## アンちゃんへの指示
1.  新しい執筆や開発を行う際は、必ず `projects/` 配下に新規フォルダを作成して作業してください。
2.  完成した成果物（note記事など）の最終版のみを `published/` にコピー・移動してください。
3.  プロジェクトが完了したら、作業フォルダを `archive/` に移動して整理してください。
