# アンちゃん（Antigravity）活用ガイド：知識ベース版

`knowledge` フォルダの情報を解析し、私（アンちゃん）とのやり取りをよりスムーズにするためのコツをまとめました。

## 1. 私を「軍師」にするための情報の渡し方
- **構造化されたDBを利用する**:
  Notionの「調査指示用DB」に情報を入れる際、ただ「〇〇について調べて」と書くよりも、**「インプットDB（過去の自分の投稿）」** と **「プロンプト（目指すべき形式）」** をセットで明示すると、精度が10倍高まります。
- **「Rules」の力を信じる**:
  新しく追加した `GEMINI.md` のルールにより、私は「無限ループ」や「パッチの勘違い」を自分で抑制するようになりました。何かトラブルが起きたときは「ルールに従って報告して」と言うだけで、原因分析モードに入ります。

## 2. 業務別の「魔法のフレーズ」
- **資料を作るとき**: 
  「1スライド1メッセージの原則で、まずは目次の承認を取ってからスライドの中身を書いて」と指示してください。無駄な長文生成を防げます。
- **GASや自動化を作るとき**: 
  「非エンジニア向けの手順もセットで」と付け加えることで、私が設定場所まで詳しく解説します。
- **リサーチをするとき**:
  「最新のWeb情報を1次ソースとして確認して」と言うことで、私の古い訓練データではなく、最新のパッチノートやニュースを取り込みます。

## 3. 収益化に向けた「資産」としての使い方
- `knowledge/raw` に放り込んだXの投稿やnoteの記事は、私にとっての「最高の参考書」です。
- 定期的に「この1週間のトレンドを分析して、noteのネタを3つ提案して」と聞くことで、`trend-watcher` と連携した精度の高い企画出しが可能になります。
