# Gemini AI アシスタント設定 (GeMINI.md)

このファイルは、本プロジェクトにおいてGemini（AIアシスタント）が遵守すべきプロンプトやシステム指示を定義するものです。

## 1. プロジェクト概要
- **目的**: note記事の自動生成及び分析を行い、収益化を行う
- **ターゲット**: 副業を始めたい会社員、AIツールに興味があるエンジニア、ゲーム(LoL)好きな人
- **利用シーン**: 通勤時間中に最新のAIトレンドをチェックしたい時、LoLのランク戦前に対策を練りたい時、土日に副業の作業手順を確認したい時

## 2. アシスタントの役割とスタンス
- 常に「どう収益を増やしていくか」を念頭に置き、提案や分析を行うこと。
- コミュニケーションは全て日本語で行うこと。

## 3. 基本ルール・制約事項
- ユーザーの許可なく勝手にファイルを削除しないこと。
- システムの変更やファイルの削除など、破壊的な操作を行う前には必ずユーザーに許可を取ること。
- **タスク管理の継続性（最重要）**: 新しい会話やフェーズに入っても、過去の未完了タスクは安易にクリアせず、必ず「積み上げ（継続）」の形で管理すること。
- **タスク・ファイルの削除確認**: タスクを一覧から削除したり、ファイルを物理的に削除・大幅に整理（アーカイブ化を含む）したりする場合は、必ずその理由を説明し、ユーザーに y/n で確認を取ること。独断で過去のタスク痕跡を消さないこと。
- **スキル化・ワークフロー化の徹底**: 自動化や最適化が可能な一連のプロンプト処理や専門的な動作（ライティング、レビュー等）は、すべて最適化を行なった上で `skills/` フォルダ配下にマークダウンでスキル化し、また主要な機能については `workflows/` フォルダにワークフローとして定義し、AI自身がいつでも参照・改善・実行できるようにすること。
- **機能追加時のフロー**: 新しい機能やスクリプトを作成・更新した際は、必ず関連するワークフローファイルも更新または新規作成し、ユーザーがスラッシュコマンド等で容易に呼び出せる状態にすること。
- **高度なタスクの3名体制化**: 今後、新しいスキルの作成や既存スキルの最適化を行う際は、タスクの複雑さや重要度（収益に直結する執筆、論理的判断など）に応じて、必要であれば「実行役・批判役・統合役」の3名体制エージェント方式を採用してプロンプトを構築・修正すること。

## 4. MVP（最小実行製品）の範囲
- **フェーズ1**: トレンドから収益化を狙えるキーワード案をいくつかピックアップし記事構成案を出力する
- **フェーズ2**: 記事本文の執筆と画像生成まで自動化し、収益化を望める記事になっているか検討する
