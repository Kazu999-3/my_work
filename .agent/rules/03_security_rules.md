# セキュリティ・ガードレイル

## ツール実行の許可ルール
ツール実行（コマンド実行、ファイル書き込み等）の許可を求める際は、必ず**日本語**で説明し、以下のリスクを推定パーセンテージ(%)で提示する。
- **秘密漏洩リスク**: パスワードや秘密鍵が外部に漏れる可能性
- **外部送信リスク**: 外部サーバーに意図しないデータが送られる可能性
- **コード実行リスク**: 悪意あるコードが勝手に動作する可能性
- **設定変更リスク**: PCや環境の設定が書き換わる可能性

## パッケージ導入の「三重審査」
AIが新しいパッケージのインストールを提案した場合、人間は以下の3点を確認してから許可（y）を出す。
1. **実在性と信頼性**: npm/Googleで検索し、スター数やダウンロード数が十分か確認する。
2. **更新頻度**: 最終更新が数年前のものは避け、メンテナンスされているか確認する。
3. **Typosquatting確認**: パッケージ名が有名なものと酷似していないかを確認する。

## 環境・設定の保護
- **ignore-scripts**: `.npmrc` にて `ignore-scripts=true` を常時設定し、インストール時の悪意あるコード実行を遮断する。
- **シークレット管理**: `.env` や機密ファイルは絶対に `git push` せず、トークン類は定期的なローテーションを検討する。
