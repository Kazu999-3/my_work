# Deep Research Protocol v3 (DRv3)

このプロトコルは、情報の網羅性、具体的数値、およびクロスリファレンスによる独自洞察を担保するためのものです。

## 1. 調査開始：検索クエリの拡張 (Expansion Phase)
リサーチ指示を受けた際、最初の一手として**5つ以上の多角的検索クエリ**を生成し、実行せよ。
- **[Q1] 公式ソース**: 最新パッチノート、開発者ブログの実数値。
- **[Q2] 統計的真実**: Lolalytics, OP.GG, U.GG 等での勝率・ピック率・バン率の推移。
- **[Q3] プロフェッショナル**: Probuilds.net, Leaguepy 等でのプロ・ハイエロのビルド順、ルーン、スキル上げ。
- **[Q4] メタ・ディスカッション**: Reddit (r/summonerschool 等), Discord, X での議論・隠れたバグ・新発見。
- **[Q5] 実技解析**: Youtube 等でのフルクリアタイム分析、特定のコンボのフレーム単位の解説。

## 2. 実効調査：多重抽出 (Extraction Phase)
検索結果の上位3〜5件に対して必ず `read_url_content` を実行し、以下の「生データ」を収集せよ。
- **数値的根拠**: ステータスの増減だけでなく、スケーリング、クールダウン、マナコストの微細な変化。
- **比較対象**: 前パッチとの比較、および競合チャンピオンとの差別化ポイント。

## 3. 分析・計算：価値変換 (Inference Phase)
収集した生データを元に、AIならではの「論理的飛躍」と「計算」を行え。
- **ripple effect (波及効果)**: 「ADが3増えた」→「Lv4時点でのジャングルキャンプ一周が約4.2秒早まる」→「スカトル戦でのレベル先行率が○%向上する」といった推論。
- **Gold Efficiency**: 推奨アイテムの変更が、ゴールド効率にどう影響し、パワースパイクを何分早めるか。

## 4. 検証：データ乖離の解明 (Conflict Resolution)
複数のソースで情報が異なる場合（例：おすすめビルドがサイトAとBで違う）、その理由を解明せよ。
- サンプル数の差、対象レートの差、特定のカウンター構成への特化など、情報の「裏」を読み解く。

## 5. アウトプット：リサーチ調査報告書の生成 (Artifact Generation)
記事執筆に入る前に、以下の構成でリサーチ報告書を作成し、ユーザーの承認を得よ。

### [報告書構成案]
1.  **エグゼクティブ・サマリー**: 調査で判明した「最大の勝機」を一言で。
2.  **ソース・マトリクス**: 参照した全URLと、抽出した主要な情報のリスト。
3.  **数値的ファクト**: パッチの変更点と、それによる「数学的な変化」の証明。
4.  **軍師の独自推論**: データの行間から読み解いた、大衆が気づいていない「勝ち筋」。
5.  **執筆方針案**: どのようなフックで読者を惹きつけ、どのような構成で価値を提供するか。

## 6. 禁止事項
- 1〜2回の検索だけで調査を終えること。
- 「一般論（例：一般にこのチャンプは機動力が高い）」で茶を濁すこと。
- 出典を明記せずに数値を提示すること。
- 調査報告書（Artifact）を作成せずに本稿の執筆を開始すること。
