# ユーザビリティ憲章 (Usability & UX Rules)

本ファイルは、AIエージェントがフロントエンド（Web UI）やアルゴリズムを実装する際に、**ユーザーの明確な指示がなくても自律的に組み込むべき「使いやすさの絶対基準」**を定めたものです。
AIはUI設計や機能実装を行う際、常にこの憲章を暗黙の要件として適用しなければなりません。

## 1. 一覧表示における絶対要件
データの一覧（例：チャンピオン辞典、マッチアップリスト、攻略ライブラリなど）を画面に表示する際は、必ず以下の機能を組み込むこと。
- **ソート機能 (Sort)**: デフォルトで「更新日（最新順）」でソート可能にすること。その他、名前順や評価順での切り替え機能を実装する。
- **絞り込み・検索 (Filter/Search)**: 大量のデータから目的のものを瞬時に見つけられるよう、カテゴリ（ロール・役割）での絞り込みや、フリーワード（チャンピオン名など）での検索フィルターを実装する。

## 2. マッチング・バランシングの絶対要件
プレイヤーのチーム分けやリソースの分配など、バランシングに関わるアルゴリズムを実装する際は、以下の要件を満たすこと。
- **均等化 (Equalization)**: プレイヤーの能力値（MMR等）のチーム合計が可能な限り均等になるよう、数学的な最適化ロジックを必ず含めること。
- **役割の尊重 (Role Preference)**: ユーザーが「やりたくない（NG）」と設定した役割やポジションには、絶対にアサインされないような強い制約（ハード制約）を設けること。

## 3. エラーとフィードバックの視認性
- **ローディングと結果**: 「画面が真っ暗になる」「ボタンを押しても反応がない」という状況を避けるため、データ取得中は必ずローディングインジケーター（スピナー等）を表示し、エラー時は人間が読んで理解できる日本語のメッセージ（トースト等）を表示すること。
