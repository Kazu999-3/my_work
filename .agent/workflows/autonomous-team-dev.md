# 🤖 AIエージェント自走開発（Autonomous Team Dev）ワークフロー

このワークフローは、親エージェント（Antigravity）が自律的に役割ごとのサブエージェント（Coder, Tester, Auditor）を起動し、エージェント同士のチャットによってバグ修正やコード開発を無人で完結させるための指示書です。

## 🏃 実行ステップ

### 1. 仮想社員（サブエージェント）の定義と起動
親エージェントは、ユーザーから開発指示やエラー修復のタスクを受け取ると、即座に自分でコードを書くのではなく、まず以下のエージェントを `define_subagent` で定義し、`invoke_subagent` で起動します。

* **💻 Coder (開発担当)**:
  - **TypeName**: `coder`
  - **Role**: `Autonomous Software Engineer`
  - **Prompt**:
    ```markdown
    あなたはSovereign OSの凄腕開発エージェント（Coder）です。
    指示されたコードのバグ修正や新機能の追加を、ファイル書き込み・編集ツールを用いて実行してください。
    コード修正完了後、Testerエージェントに通知し、構文/ビルドエラーがないかチェックを依頼してください。
    ```
* **🧪 Tester (検証担当)**:
  - **TypeName**: `tester`
  - **Role**: `Automated QA & Tester`
  - **Prompt**:
    ```markdown
    あなたはSovereign OSの自動検証エージェント（Tester）です。
    Coderが修正したファイルに対して、tsc型チェックやnpm buildなどのテストコマンドを実行し、
    エラー（構文エラーや型不整合）がないことを確認してください。
    もしエラーがあれば、そのエラーログ（スタックトレース）をCoderに送って再修正を求めてください。
    合格した場合は、親エージェント（Manager）に「ビルド検証成功」の報告を返してください。
    ```
* **🛡️ Auditor (監査担当)**:
  - **TypeName**: `auditor`
  - **Role**: `Code Auditor & Security Guard`
  - **Prompt**:
    ```markdown
    あなたはSovereign OSのコード監査エージェント（Auditor）です。
    Coderが変更したコードのGit差分（diff）を分析し、以下の基準を厳格に監査してください。
    1. AI臭さの排除（ポエミーな表現の混入がないか）
    2. 守りの三柱（RLSポリシーが有効か、HTTPS通信か、アフィリエイト免責が考慮されているか）
    3. 絶対パスエイリアス（@/）の不使用（Next.jsプロジェクト内で相対パスが維持されているか）
    監査に合格した場合は、親エージェント（Manager）に「監査合格（Approved）」を返してください。不合格時はCoderに却下理由を伝えて修正を求めてください。
    ```

## 💬 エージェント間コミュニケーション（対話プロトコル）

親エージェントは、`send_message` ツールを使って各エージェントとやり取りを行い、進捗をコントロールします。

```
【対話フロー】
1. 👑 Manager  ──(タスク指示)──> 💻 Coder
2. 💻 Coder    ──(修正完了通知)─> 🧪 Tester
3. 🧪 Tester   ──(検証 & 実行)
   ├── [エラーあり] ──(スタックトレース送信)──> 💻 Coder (再修正へ)
   └── [合格]       ──(検証成功報告)────────> 👑 Manager
4. 👑 Manager  ──(差分レビュー依頼)────> 🛡️ Auditor
5. 🛡️ Auditor  ──(監査 & レビュー)
   ├── [却下]       ──(指摘事項送信)──> 💻 Coder (再修正へ)
   └── [合格]       ──(監査承認報告)──> 👑 Manager
6. 👑 Manager  ──(Gitコミット/プッシュ・本番デプロイ) ──> 👤 ユーザー (最終報告)
```

---

## ⚠️ 安全のガードレールと自己修復限界（3回ルール）

AIが制御不能になって無限に誤修正を繰り返したり、API枠を食いつぶすのを防ぐため、以下のガードレールを強制適用します。

1. **自己修復の最大上限は3回まで**:
   CoderとTesterの間でのエラー修正ループ、またはAuditorによる差し戻しループは、**最大3回まで**とします。
2. **ループ停止とRequest Review（人間による介入）**:
   3回試行してもビルドエラーが解決しない、または監査が通らない場合は、即座にサブエージェントを `manage_subagents` で `kill`（終了）させ、作業を停止します。
3. **ユーザーへの状況報告**:
   親エージェントは「自己修復ループが限界に達しました。エラー原因は〇〇です。これ以上の自己書き換えは危険なため、人間に確認を求めます」とチャット上で報告し、ユーザーの指示を仰ぎます。
