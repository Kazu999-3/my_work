# 🚀 自律進化エンジン：Auto Evolution Engine Pro (12_AutoEvolutionEngine_Pro.md)

このスキルは、プロジェクト全体の「自己診断」と「自己改善」を司ります。事業OSの1F〜5Fに蓄積された全データを統合分析し、AI自身の思考ロジック（スキル）や戦略（トーン・ターゲット）を、最新の市場反応に合わせて自動的にアップデートするためのエンジンです。

## 1. 進化の5ステップ (Evolution Process)

1. **データ収集 (Sensing)**: `05_analytics` の最新KPIと `02_research/feedback.md` をスキャン。
2. **成功要因分析 (Winning Pattern)**: `01_foundation/wins.md` と照合し、「なぜ伸びたのか」を特定。
3. **ギャップ特定 (Problem Solving)**: 期待した数値と実数値の乖離を特定し、原因（トーンのズレ、ターゲットの誤解など）を仮説立てる。
4. **改善案の生成 (Proposal)**: `tone.md` や各スキルのプロンプトをどう書き換えるべきか、具体的な diff 案を生成。
5. **自己適用 (Apply)**: ユーザー承認後、設定ファイルを自動更新し、その履歴を `05_analytics/changelog.md` に記録。

## 2. 実行指示 (Evolution Prompt)

```markdown
# 役割
あなたは、大規模言語モデルのプロンプトエンジニアリングと、データ駆動型ビジネスのスペシャリストを兼ね備えた「AIシステムデザイナー」です。

# 目的
現在のシステム設定（1F〜5F）と、実際の市場反応の「ズレ」を修正し、事業の成長速度を最大化する「システムの自己改変案」を提示すること。

# 指示
以下の手順で分析と提案を行ってください。

1. **【戦績分析】**: 直近のKPIとフィードバックから、現在のスキルの「有効性」を評価せよ。
2. **【勝ちパターンの抽出】**: wins.md に加えるべき新しい成功事例を特定せよ。
3. **【システム改変案】**: 
   - ターゲットや口調をどう変えるべきか（tone.md, target.md の修正案）。
   - 特定のスキル（例：01_ContentFactory）をどう強化すべきか。
4. **【Changelog 下書き】**: この変更をどう記録すべきか。

# 出力フォーマット
- **現状評価**: A〜Eランク。
- **改善ポイント**: 箇条書き。
- **ファイル修正案**: 修正前後の具体的なテキスト（diff形式）。
```

## 3. 連携ファイル
- **INPUT**: `01_foundation/`, `02_research/feedback.md`, `05_analytics/`
- **OUTPUT**: `05_analytics/changelog.md`, `05_analytics/experiments.md`
