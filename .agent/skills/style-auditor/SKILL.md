# スタイル監査人 (style-auditor)

このスキルは、作成されたドラフト（note記事、戦略ガイドなど）を `ANTIGRAVITY.md` および `prohibited_patterns.md` の基準で監査し、即座に修正案を提示します。

## 監査の核心
1. **「AI臭さ」の排除**: 定型句（「いかがでしたでしょうか」等）や、過剰に丁寧な言い回しを削除。
2. **「むだな太字」の乱用停止**: `**` の多用を避け、スッキリとした構成にする。
3. **設計士（Designer）の視点**: タイピストのような受動的表現を、能動的・戦略的な提案へとリライト。
4. **文字数要件**: note記事などの目標文字数（3000文字以上）の成否を確認。

## 実行手順

### 1. 高速スキャン実行
まずは以下のコマンドで、物理的な禁止表現と文字数をチェックしてください。
`python d:/my_work/05_system/scripts/style_auditor.py <file_path>`

### 2. セマンティック監査（AIによる深層評価）
スクリプトの結果を踏まえ、以下のプロンプトで内容をリライト案として提示します：
「修正前後の比較を `diff` または code block で提示せよ。特にむだな `**` は排除し、人間味のある断定的な語り口（Designer視点）に修正すること。」

## レポート形式
修正箇所は以下のように、修正前（❌ Before）と修正後（✅ After）の対比で出力してください。

```markdown
### ❌ Before
〜〜〜 **絶対に** お役に立てれば幸いです。 〜〜〜

### ✅ After
〜〜〜 この仕組みを武器に、自由を勝ち取ってください。 〜〜〜
```
