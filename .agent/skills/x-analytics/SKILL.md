# 📈 X Analytics

## 目的
X(Twitter)アカウントのフォロワー推移と投稿パフォーマンスを分析し、拡散戦略の効果測定と次回施策の提案を行う。

## 使用タイミング
- `/pdp-operational-cycle` ワークフローのStep 1として自動起動
- 収益番頭エージェント(Treasurer)からの呼び出し
- ユーザーが「Xの反応を分析して」と依頼した時

## 入力パラメータ
| パラメータ | 必須 | 説明 | 例 |
|:---|:---|:---|:---|
| `account` | ❌ | アカウント名（デフォルト: @JggapggLol） | `@JggapggLol` |
| `period` | ❌ | 分析期間（デフォルト: 7日） | `30days` |

## 実行手順

### Step 1: 公開情報の取得
```
browser_subagent を使用:
1. https://x.com/{account} にアクセス
2. フォロワー数を取得
3. 直近の投稿一覧（リプライ数・リポスト数・いいね数・表示回数）を取得
```

### Step 2: エンゲージメント分析
```markdown
## 📈 X パフォーマンスレポート ({日付})

### アカウント概要
- フォロワー: {N}名（前回比 +{X}）
- 直近{期間}の投稿数: {Y}本

### 投稿パフォーマンス Top 5
| # | 投稿要約 | 表示 | いいね | RT | ER(%) |
|:--|:---|:---:|:---:|:---:|:---:|
| 1 | {要約} | {表示数} | {いいね} | {RT} | {ER} |

### インサイト
- 🔥 最もエンゲージが高い投稿タイプ: {分析}
- 📅 最適な投稿時間帯: {分析}
- 💡 次回のアクション: {提案}
```

## 出力先
- `03_FACTORY/assets/analytics/x_report_{date}.md`

## 制約
- Xのログインガードが強力なため、公開情報の取得を優先
- API（X API v2）が利用可能な場合はそちらを優先使用
- レート制限に注意（15分あたり最大15リクエスト）
