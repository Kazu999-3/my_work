# 📊 Note Analytics

## 目的
noteに公開された記事の反応データ（PV・スキ・コメント）を収集・分析し、次回コンテンツの方向性を決定するためのインサイトを提供する。

## 使用タイミング
- `/pdp-operational-cycle` ワークフローのStep 1として自動起動
- 収益番頭エージェント(Treasurer)からの呼び出し
- ユーザーが「noteの反応を分析して」と依頼した時

## 実行手順

### Step 1: noteダッシュボードへアクセス
```
browser_subagent を使用:
1. https://note.com/dashboard にアクセス
2. ログイン（.envからクレデンシャルを取得）
3. 記事一覧ページから全記事のデータを取得
```

### Step 2: データ抽出
各記事について以下を抽出:
- 記事タイトル
- 公開日
- PV数
- スキ数
- コメント数
- 有料/無料の区分
- 価格（有料の場合）

### Step 3: パフォーマンス分析
```markdown
## 📊 note パフォーマンスレポート ({日付})

### サマリー
- 総記事数: {N}本
- 合計PV: {X}
- 合計スキ: {Y}
- PV/記事 平均: {Z}

### 記事別ランキング（PV順）
| # | タイトル | PV | スキ | CVR(スキ/PV) |
|:--|:---|:---:|:---:|:---:|
| 1 | {タイトル} | {PV} | {スキ} | {%} |

### インサイト
- 🔥 最もPVが高いカテゴリ: {分析結果}
- 📈 成長トレンド: {前回比}
- 💡 次回への提言: {具体的なアクション}
```

## 出力先
- `03_FACTORY/assets/analytics/note_report_{date}.md`

## 制約
- noteのログイン状態が必要（Playwrightを通じたブラウザ操作）
- APIが存在しないためスクレイピングに依存（構造変化に注意）
- 取得失敗時は直前のキャッシュデータを使用し `⚠️ キャッシュデータ` と明記
