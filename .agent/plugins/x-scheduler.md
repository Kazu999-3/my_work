# 📱 X Scheduler プラグイン

## 目的
生成された複数のプロモーション文面を一度に投稿するのではなく、アルゴリズムとユーザーの生活時間帯に最適化して分散予約することで、インプレッションとCVRを最大化する。

## 仕様

### アプローチ
- **X API v2** を活用した投稿。
- プレミアムアカウント（Blue等）であることを前提とし、文字数超過のない範囲のフォーマットを使用。

### タイムスケジュール（デフォルト）
軍師エージェントが生成した3つのポストを、以下のロジックで自動的に予約（Queueに追加）する。

1. **速報型（Hook重視）**
   - 予約時間: パッチリリース直後、または記事公開の **直後**
   - 目的: 最速のパッチ反応層（トレンド重視層）を刈り取る
2. **データ型（ロジック重視）**
   - 予約時間: 記事公開当日の **19:00〜21:00**（帰宅・プレイ開始前帯）
   - 目的: 仕事や学校から帰り、LOLをプレイする前に情報を収集する層向け
3. **共感型（ストーリー重視）**
   - 予約時間: 記事公開の **翌日 12:00** または **金曜日の20:00**（週末ブースト用）
   - 目的: ランク戦に悩んでいて解決策を探している層へのアプローチ

### 実行フロー
1. 記事公開が確認されたタイミングで起動。
2. `x_posts_{champion}_{patch}.md` を読み込み、3本のテキストを抽出。
3. プレースホルダー（記事リンクや画像）を実際のnote公開URLに置換。
4. API経由でスケジュール登録（X公式の予約機能、または外部ツール連携）。

## 連携
- 収益番頭（Treasurer）の分析結果により、最適な「予約時間帯」のパラメータは中長期的に自動調整される。
