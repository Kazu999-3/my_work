---
description: 昨日の作業報告書を自動生成する
---

ユーザーが `/daily-report` と入力した際、AIアシスタントは以下の手順で昨日の作業報告書を自動生成してください。

1. **記憶の振り返り**
   - 直近の会話や作業履歴から、昨日一日で行った作業内容を**可能な限り詳細に抽出**します。
   - 加えて、以下のコマンドを実行して、スプレッドシートの Tasks シートから最近（昨日など）「完了」になったタスクを取得し、これも報告書に組み込みます。
     ```bash
     python d:\my_work\apps\youtube_manager\src\task_manager.py list --status "完了"
     ```
   - 単なる概要にとどまらず、**どのような意図**で、**具体的にどのファイル**を、**どのように変更したか**といった背景や経緯を含めてください。
   - 以下の項目を含むように情報を整理してください。
     - **完了したタスク**: 取り組んだプロジェクト名、Tasksシートから取得した完了タスク情報、具体的な作業手順、技術的な設計、最終的な成果物。
     - **発生した課題やエラーとその対応**: 直面したエラーの詳細、解決に至ったアプローチや論理展開。
     - **次回のタスク・提案**: 今後取り組むべき作業に加え、より良くするためのアイデアや収益化に繋がる提案を具体的に。
   - 全体として、質の高いブログ記事やNoteの執筆ネタとしてそのまま流用できるレベルで、読みやすく論理的な構成で記述してください。

2. **出力先ディレクトリの確認**
   - `d:\my_work\outputs\report` ディレクトリが存在するか確認し、無い場合は作成してください。

3. **Markdownファイルとしての保存**
   - ファイル名は **実行した日の日付**（形式: `YYYY-MM-DD.md`）とします。
   - 整理した詳細な報告書テキストを、対象のファイル（`d:\my_work\outputs\report\YYYY-MM-DD.md`）に保存します。

4. **完了報告**
   - ファイルの保存が完了したら、ユーザーに保存先パスと簡単なプレビューを提示し、報告します。
