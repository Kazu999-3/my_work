# KTM Bot 拡張計画: チーム分け自動化プロトコル

## 1. プロジェクト概要
Discordでのカスタム募集からチーム分け、スプレッドシートへのデータ反映までを完全に自動化し、管理者の手動操作をゼロにする。また、Discordイベント参加者の自動通知など運用の自動化を拡充する。

## 2. MVP（最少構成）の範囲
- 募集枠（10名）が埋まった瞬間にトリガーを起動。
- 参加確定した10名の名前を「対戦入力」シートのA列に即時反映。
- 続けてチーム分け（バランサー）を実行し、シートを更新。
- 結果を Discord に自動投稿し、メンションで通知。
- **定期通知プロトコル (イベント興味あり抽出)**:
  - 毎週金曜日と土曜日の20:00(JST)に、直近48時間以内に開始予定でタイトルに「【定期】」が含まれるDiscordイベントの「興味あり」ユーザーを自動抽出。
  - 抽出結果をチャンネルID `1487077567939743995` へ綺麗なEmbed（埋め込み）カード形式で送信する。
- **定期イベント自動作成プロトコル**:
  - 毎週土曜日の21:00(JST)に、翌週の土曜日21:00開始の単発イベントを2つ自動作成する。
    1. タイトル: `【定期】シルバー以下カスタム` / 場所: `オンライン`
    2. タイトル: `【定期】ゴルプラ以下カスタム` / 場所: `オンライン`
  - 重複防止: 同一開始予定日時・同一タイトルのイベントがすでに存在する場合は、作成を自動でスキップする。
  - 全体告知: イベント作成後、それぞれのイベントリンク付きで指定チャンネルへ `@everyone` 全体メンション告知メッセージを自動送信する。

## 3. 技術スタック
- **Backend**: Google Apps Script (GAS)
- **Frontend/Middleware**: Cloudflare Workers (Discord Bot) + Cron Triggers
- **Database**: Google Sheets
