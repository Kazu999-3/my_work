## はじめに

KTM Bot は、LoL カスタムマッチの **募集 → レーン決定 → チーム分け → 戦績記録** を一気通貫で管理するDiscord Botです。

全体のフローはこうなっています：

```
① /recruit で人を集める
② /lane  で希望レーンを事前登録しておく
③ /balance でAIが公平にチーム分けする
④ 試合後にボタンを押すだけで戦績が自動記録される
```

## 1. メンバー募集コマンド `/recruit`

### 基本の使い方

チャット欄に `/recruit` と入力するだけ。募集パネルが投稿されます。

### オプション一覧

| オプション名 | 必須 | 説明 | 例 |
|:---|:---:|:---|:---|
| `mode` | × | ゲームモード。`カスタム`（10人）/ `ノーマル`（5人）/ `ARAM`（5人）| カスタム |
| `time` | × | 開始予定時刻 | 21:00〜 |
| `max` | × | 最大人数（デフォルト：カスタム=10、ノーマル/ARAM=5）| 10 |
| `memo` | × | 一言メモ | 初心者歓迎！ |
| `player1`〜`player5` | × | 最初からメンバーを追加しておける | @ユーザー |

### 募集パネルのボタン操作

募集が投稿されると、メンバー全員が操作できるボタンが表示されます。

| ボタン | 誰が押せるか | 何が起きるか |
|:---|:---|:---|
| ✋ **どこでも参加** | 誰でも | 参加者リストに追加される |
| 🛡️ Top / ⚔️ Jg / 🧙 Mid / 🏹 Adc / 🩹 Sup | 誰でも（ノーマルモード時） | 指定ロールで参加 |
| ⏳ **カスタム待機** / 👁️ **観戦希望** | 誰でも | 試合には参加せず待機枠に入る（もう一度押すと解除） |
| 🏃 **離脱** | 誰でも | 参加・観戦から抜ける |
| 🚀 **10人に拡張** | 募集主のみ | ノーマル/ARAMの5人募集をカスタム10人に切り替える |
| 👥 **代理追加** | 募集主のみ | Discord上でユーザーを選択して代わりに参加させる |
| 📢 **一括連絡** | 募集主のみ | メッセージを書くと参加者全員にメンションが飛ぶ |
| 🚩 **募集終了** | 募集主のみ | 募集を締め切る |

### 自動締め切り

定員に達した瞬間、自動的に **「⚔️ メンバー確定」** に変わり、参加者全員にメンションで通知が飛びます。
確定後は「🏆 チーム分け実行」ボタンが出現するので、そのまま押せば即座にチーム分けが始まります。

## 2. レーン設定コマンド `/lane`

### なぜ設定が必要？

チーム分けの際、あなたの **希望レーン**・**絶対NG なレーン**・**こだわり度** をBotが参照し、なるべく全員が納得できる配置を自動計算します。**事前に設定しておくことで、毎回の体験が格段に良くなります。**

### 使い方（2つの方法）

#### 方法A: スラッシュコマンドで直接指定

```
/lane main:JG sub:TOP ng1:SUP weight:1
```

| オプション | 必須 | 説明 | 入力値 |
|:---|:---:|:---|:---|
| `main` | ✅ | メインレーン | `TOP` / `JG` / `MID` / `ADC` / `SUP` / `ALL` |
| `sub` | × | サブレーン（2番目に得意） | 同上 |
| `ng1` | × | NGレーン1（行きたくないレーン） | 同上 |
| `ng2` | × | NGレーン2 | 同上 |
| `weight` | × | こだわり度（後述） | `1` / `2` / `3` |
| `allow_higher` | × | 格上プレイヤーとの対面を許容するか | `true` / `false` |

#### 方法B: モーダル（入力フォーム）で設定

`/lane` をオプションなしで実行すると、入力フォームが開きます。
各欄に記入して送信するだけでOKです。

### こだわり度について

`weight`（こだわり度）は、チーム分け時にあなたのメインレーンをどれくらい優先するかを決める重要な設定です。

| 値 | 意味 | おすすめユーザー |
|:---:|:---|:---|
| **1 (絶対)** | 「メインレーン以外はやりたくない」最優先で配置される | JG/SUP専など、特定ロールに特化している人 |
| **2 (通常)** | バランスよく配置される（デフォルト） | 大半の人はこれでOK |
| **3 (柔軟)** | 「どこでもいいよ」チーム全体の最適化が優先される | フィル（サブ枠）にも抵抗がない人 |

### 「ALL」の意味

メインレーンに `ALL` を設定すると、「どのレーンでも均等に扱ってOK」という意味になります。こだわりがない人やフィル希望の人向け。

## 3. チーム分けコマンド `/balance`

### 基本の使い方

ボイスチャンネル（VC）に **10名** が接続した状態で `/balance` を実行します。
VCのメンバーリストから自動的にプレイヤーを取得し、チーム分けを実行します。

### 募集パネルからの実行

定員に達した募集パネル上の「🏆 チーム分け実行」ボタンからも実行可能です。この場合、VCではなく募集パネルの参加者をもとに計算します。

### チーム分け結果の見方

```
⚔️ チーム分けの結果 (KTM Balancer)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🟦 Team A (Blue)                🟥 Team B (Red)
TOP  プレイヤーA (1350)         TOP  プレイヤーF (1280)
JG   プレイヤーB (1400)         JG   プレイヤーG (1420)
MID  プレイヤーC (1500)         MID  プレイヤーH (1480)
ADC  プレイヤーD (1250)         ADC  プレイヤーI (1300)
SUP  プレイヤーE (1200)         SUP  プレイヤーJ (1180)

勝率平準化・ユーザー分離ロジック適用済み
```

- 名前の横の **カッコ内の数字** がそのロールでの **MMR（内部レート）** です。
- Blue/Red はランダムに入れ替わるので、陣営による有利不利はありません。

### 勝敗報告

チーム分けの結果には、2つのボタンが表示されます：

| ボタン | 動作 |
|:---|:---|
| 🟦 **BLUE 勝利** | Blue チームの勝利を報告 |
| 🟥 **RED 勝利** | Red チームの勝利を報告 |

ボタンを押すと：
1. 試合結果が **自動的にスプレッドシートに記録** される
2. 各プレイヤーの **MMRが更新** される
3. #戦績板 に結果が投稿される
4. **次の試合のチーム分けが自動的に開始** される（連戦対応）

> ⚠️ 勝敗報告ボタンは、`/balance` を実行した本人のみ押せます。

## 4. マッチングシステムの仕組み（詳細）

KTM Balancer は、ただのランダムシャッフルではありません。**以下の要素を全て考慮した上で、最も公平な組み合わせを自動的に選び出します。**

### 4-1. MMR（内部レート）

各プレイヤーに **ロールごとの独立したMMR** を持っています（初期値: 1200、最高ランクに応じた圧縮値からスタート）。

- JGのMMRが1400でも、TOPのMMRは1100ということがあり得ます。
- チーム分け時は **そのロールに配置された場合のMMR** が使われます。

MMRの増減は以下の要素で決まります：
- **Eloレーティング**: 勝敗と対面相手のMMRの差から計算（K=48）
- **KDAボーナス**: KDA（キル・デス・アシスト）を元に最大 ±20 の加減点補正
- **ランク収束引力**: まだ試合数が少ないプレイヤーは、公式最高ランクの適正レートに引き寄せられる補正が入る
- **プレースメント倍率**: 5試合未満は **3倍**、10試合未満は **2倍** に変動量がブーストされる（早く適正レートに到達するため）
- **対戦回数倍率**: 同じ対面相手との対戦回数が少ないうちは変動量が最大 **1.5倍** になり、対戦を重ねると 1.0 に収束する

### 4-2. レーン配置の最適化

全プレイヤーの全ロール組み合わせ（約14,400通り × C(10,5) = 約252通り）を計算し、以下のペナルティが最も少ない組み合わせを選びます。

| 条件 | ペナルティの重さ | 説明 |
|:---|:---:|:---|
| **NGレーンへの配置** | ★★★★★ | 200万ポイント。絶対に避けられる |
| **メインでもサブでもないレーンへの配置** | ★★★★ | 50万〜。Pity値で加算 |
| **サブレーンへの配置** | ★★ | 2万〜。Pity値で加算 |
| **メインレーンへの配置** | なし | ペナルティ0。理想状態 |
| **こだわり度1（絶対）のプレイヤーがサブレーンに配置** | ★★★★★ | 通常の50倍のペナルティ |
| **こだわり度3（柔軟）のプレイヤーがサブレーンに配置** | ★ | 通常の1/4のペナルティ |
| **専門職（JG/SUP/ADC）がメイン以外に配置** | ★★★★ | 追加倍率（×2〜×3）がかかる |
| **新規プレイヤー（未ランク＆3戦未満）がJG/MIDに配置** | ★★★★ | 100万ポイント。経験が必要なロールを初心者に振らない |

### 4-3. Pity（不運度）システム

「何回も続けてサブレーンに行かされる…」という不満を防ぐ仕組みです。

| 配置されたレーン | Pity の変化 |
|:---|:---|
| メインレーン | **0 にリセット** |
| サブレーン | **+2** |
| メインでもサブでもないレーン | **+5** |
| 試合に参加できなかった（カスタム待機） | **+10** |

Pityが高いプレイヤーほど、次の試合でメインレーンに配置されやすくなります。
つまり **「我慢した回数」が自動的に考慮される** ということです。

### 4-4. チーム戦力の均衡化

レーン配置が決まった後、以下の条件でチーム間のバランスも最適化します。

- **ロールごとのMMR差**: 各レーンの対面同士のMMR差が小さいほど良い
- **チーム全体のMMR合計差**: 総合力が近くなるように調整
- **勝率の均衡**: 直近の勝率が偏らないようにする。勝率1位と最下位のプレイヤーは同じチームに入りやすい
- **初心者の分散**: 新規プレイヤーが片方のチームに偏らない
- **対戦履歴の回避**: 直近5試合で同じレーンで対面したペアは避ける
- **格上対面の保護**: MMR差が600以上ある場合、メインレーン以外での対面にはペナルティが加算される

### 4-5. 11人以上の場合（Pityによる選出）

10人を超える参加者がいた場合、**Pity（不運度）が高い順** に優先的に試合メンバーに選ばれます。
試合に参加できなかったメンバーには「カスタム待機」枠が割り当てられ、Pityが+10されるため、次の試合では優先的に出場できます。

## 5. 戦績確認コマンド `/stats`

### 使い方

```
/stats
```

実行すると **#戦績板** チャンネルにあなたの戦績が投稿されます。

### 表示される情報

| 項目 | 内容 |
|:---|:---|
| 🏆 総合 | 合計試合数・勝ち数・勝率 |
| 🕒 直近5試合 | 🟦（勝利）/ 🟥（敗北）のアイコン列 |
| 🏮 不運度 (Pity) | 現在のPity値。高いほど次にメインレーンに行ける確率UP |
| 📍 ポジション別 | TOP/JG/MID/ADC/SUP ごとのMMR・試合数・勝率 |

## 6. よくある質問（FAQ）

### Q: レーン設定は毎回やるの？
**A: いいえ。一度設定すれば記録されます。** 変更したい時だけ `/lane` を再実行すればOKです。

### Q: MMRはどうやったら上がるの？
**A: 試合に勝つことです。** 特に自分より強い相手に勝つと大きく上がります。KDAも多少影響します。

### Q: なんで毎回同じレーンじゃないの？
**A: 10人全員の希望を完全に通すのは不可能なためです。** ただし、こだわり度を `1（絶対）` に設定すれば、ほぼ確実にメインレーンに配置されます。

### Q: チーム分けが気に入らない場合は？
**A: 「🏆 チーム分け実行」ボタンをもう一度押してください。** Blue/Red のランダム入替があるため、結果が変わる場合があります。

### Q: 新しいメンバーだけどレートが極端に高い/低い…
**A: 最初の10試合はプレースメント期間です。** 5試合未満は3倍、10試合未満は2倍の変動幅ブーストがかかるため、すぐに適正レートに収束します。

### Q: カスタム待機って何？
**A: 11人以上集まった時に、試合に出られなかった人のための枠です。** 待機したぶんPityが+10されるので、次は優先的にメンバーに選ばれます。
