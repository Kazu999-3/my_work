# フォルダ一覧と命名規則 (DirectoryRules.md)

このファイルは `d:\my_work` 配下のディレクトリ構成と、各フォルダへのファイル格納ルール・命名規則を定義するものです。

## 1. ディレクトリ構成と役割
（※ 全体のツリー構造はルート直下の `ディレクトリ構成.md` でも管理します）

- **`skills/`**: スキルの保管庫
  - AIが特定の役割を果たすための設定ファイル（PromptOptimizer.mdなど）を格納します。
- **`apps/`**: 作成した各アプリの保管庫
  - 例: `note_generator/` (note記事作成アプリ本体のプログラムなど)
- **`outputs/`**: 作成した制作物（note記事など）の保管庫
  - AIが自動生成した構成案、テキスト本文、画像ファイルなどを保存します。
- **`knowledge/`**: Obsidian等に格納するための知識（ナレッジ）保管庫
  - 調査メモ、トレンド情報の蓄積、後から振り返るためのデータなどを格納します。
- **`apps_portfolio/`**: 作成したアプリやスキルのポートフォリオ
  - 完成したアプリの概要図、説明資料、公開用の紹介文などをまとめます。
- **`rules/`**: グローバルシステムルール
  - ディレクトリの運用ルールやファイルの命名規則（本ファイルなど）を格納します。

## 2. ファイルの命名規則
- **マークダウンファイル（ドキュメント類）**:
  - 原則として「PascalCase（単語の頭文字を大文字）」＋ `.md` とします。
  - 例: `PromptOptimizer.md`, `DirectoryRules.md`
- **スクリプト・コードファイル**:
  - ディレクトリ・ファイル名ともに「snake_case（小文字アンダースコア区切り）」とします。
  - 例: `trend_analyzer.py`, `article_generator.js`
- **出力される最終制作物**:
  - `YYYYMMDD_記事タイトル略称.md` のように、日付をプレフィックスとして付与します。
  - 例: `20260302_ai_副業_trend.md`

## 3. 格納ルール
- 新しいツールやAIシステム（スキル）を作成した場合は、必ず `skills/` に追加し、どこからでも呼び出せるようにすること。
- システムやアプリが自動吐き出しする成果物は `outputs/` のみに集約し、プログラム本体（`apps/`）とは分けること。
- 情報収集などで得た「再利用可能な知見」は、必ず `knowledge/` にMarkdown形式で蓄積すること。
