# 🩹 自己修復プロポーザル

アンちゃんズ AdminAn (System Architect) です。
エラーメッセージとトレースバックから、`OmniAgent` クラスに `harvest_assets` というメソッドが存在しないことが原因で `AttributeError` が発生していることが明確に分かります。

トレースバックの該当箇所：
```
  File "D:\my_work\02_intelligence\intelligence\omni_agent.py", line 145, in run_cycle
    self.harvest_assets()
    ^^^^^^^^^^^^^^^^^^^
AttributeError: 'OmniAgent' object has no attribute 'harvest_assets'
```
ソースコードの `run_cycle` メソッド内、ステップ9で `self.harvest_assets()` が呼び出されています。
```python
            # 9. AdminAn: 資産収穫 (Asset Harvesting)
            self.log("🧺 [AdminAn] Step 9/12: 高評価ドラフトの資産化...")
            self.harvest_assets() # ここでエラー
```
しかし、`OmniAgent` クラスの定義全体には `harvest_assets` メソッドの実装が見当たりません。

このステップのコメント「高評価ドラフトの資産化」から、このメソッドの意図が推測できます。自律商社が生成したコンテンツの中で、特に評価の高いものを「資産」として永続的に保存する機能でしょう。

**修正案:**

`OmniAgent` クラスに `harvest_assets` メソッドを追加し、高評価のドラフトファイル（`03_factory/daily_posts` 内の `draft_*.md` ファイル）を、資産ディレクトリ（`01_spirit/assets`）にコピーするロジックを実装します。資産ファイル名は、他の資産管理ロジック（`assemble_products` メソッドが `evergreen_*.md` を参照している箇所）と整合性を保つため、`evergreen_` プレフィックスを使用します。

---

### 修正手順

1.  **`omni_agent.py` ファイルを開きます。**
2.  **`OmniAgent` クラス内、既存のメソッドのどこか（例えば `check_asset_candidates` メソッドの直後あたり）に以下の `harvest_assets` メソッドを追加します。**

### 修正コード（追加）

```python
    def harvest_assets(self):
        """
        [AdminAn] 高評価ドラフトを「資産」として収集し、永続化する。
        スコアが閾値以上のドラフトを `01_spirit/assets` にコピーする。
        """
        self.log("🧺 [AdminAn] 高評価ドラフトの資産化プロセスを開始します。")
        
        draft_dir = ROOT_DIR / "03_factory" / "daily_posts"
        asset_dir = ROOT_DIR / "01_spirit" / "assets"
        asset_dir.mkdir(parents=True, exist_ok=True) # 資産ディレクトリが存在しない場合は作成

        harvested_count = 0
        score_threshold = 80 # 例: 80点以上のドラフトを資産化の対象とする

        # daily_posts ディレクトリ内のドラフトファイルを検索
        for draft_file in draft_dir.glob("draft_*.md"):
            try:
                # ファイル名からスコアを抽出 (例: draft_20231026_1030_s95.md -> 95)
                # ファイル名が `draft_YYYYMMDD_HHMM_s<SCORE>.md` の形式であることを前提とする
                score_part = draft_file.stem.split('_s')[-1] # stemで拡張子を除去
                score = int(score_part)

                if score >= score_threshold:
                    # 資産ファイル名を作成 (例: draft_20231026_1030_s95.md -> evergreen_20231026_1030_s95.md)
                    new_asset_name = f"evergreen_{draft_file.name.replace('draft_', '')}"
                    new_asset_path = asset_dir / new_asset_name
                    
                    import shutil
                    # 元のドラフトを資産ディレクトリにコピー
                    shutil.copy(draft_file, new_asset_path)
                    self.log(f"   ✅ スコア {score} のドラフトを資産化しました: {new_asset_name}")
                    harvested_count += 1
                else:
                    self.log(f"   ℹ️ スコア {score} のドラフトは資産化対象外です ({score_threshold}点未満): {draft_file.name}")
            except ValueError:
                self.log(f"   ⚠️ ドラフト {draft_file.name} のスコアを解析できませんでした。スキップします。")
            except Exception as e:
                self.log(f"   ❌ ドラフト {draft_file.name} の資産化中に予期せぬエラー: {e}")
        
        if harvested_count > 0:
            self.log(f"🧺 [AdminAn] {harvested_count} 件の高評価ドラフトを資産として収穫しました。")
        else:
            self.log("🧺 [AdminAn] 資産化対象となる高評価ドラフトは見つかりませんでした。")

```

### 修正の根拠

*   **`AttributeError` の解消:** `harvest_assets` メソッドを `OmniAgent` クラスに定義することで、`self.harvest_assets()` の呼び出しが有効になります。
*   **機能の意図の実現:** `run_cycle` 内のコメント「高評価ドラフトの資産化」に基づき、ドラフトファイルからスコアを抽出し、閾値以上のものを資産として保存するロジックを実装しました。
*   **既存システムとの整合性:**
    *   `assemble_products` メソッドが `evergreen_*.md` ファイルを収集しているため、資産ファイル名に `evergreen_` プレフィックスを使用します。
    *   `harvest_assets()` が `await` なしで呼び出されているため、追加するメソッドも同期関数として実装します。
    *   資産格納先を `01_spirit/assets` とすることで、`01_spirit` 配下の重要ファイルとして管理されます。

この修正により、エラーが解消され、自律商社システムの資産収穫プロセスが正常に機能するようになります。
