### 修正案

`OmniAgent` クラスに、以下の `harvest_assets` メソッドを追加してください。クラス内のどの位置でも構いませんが、例えば `assemble_products` メソッドの直前や直後あたりが自然でしょう。

```python
# OmniAgent クラス内に以下のメソッドを追加

    def harvest_assets(self):
        """
        [AdminAn] Step 9/12: 高評価ドラフトの資産化
        daily_posts フォルダから高スコアのドラフトを選別し、
        永続的な資産として assets フォルダに格納します。
        Notion への同期も行います。
        """
        self.log("🧺 [AdminAn] 高評価ドラフトの資産化処理を開始します...")
        draft_dir = ROOT_DIR / "03_factory" / "daily_posts"
        asset_dir = ROOT_DIR / "01_spirit" / "assets"
        asset_dir.mkdir(parents=True, exist_ok=True) # 資産格納ディレクトリを確実に作成

        # 高評価の閾値 (例: 85点以上のドラフトを資産化)
        MIN_SCORE_FOR_ASSET = 85 
        
        harvested_count = 0
        for draft_file in draft_dir.glob("draft_*.md"):
            try:
                # ファイル名からスコアを抽出 (例: draft_20231026_1030_s95.md)
                parts = draft_file.stem.split('_')
                if len(parts) >= 4 and parts[-1].startswith('s'):
                    score_str = parts[-1][1:]
                    score = int(score_str)
                    
                    if score >= MIN_SCORE_FOR_ASSET:
                        # 資産化する新しいファイル名 (例: evergreen_20231026_1030.md)
                        # 日時情報のみ残し、スコア部分は資産名からは除外
                        new_asset_name = f"evergreen_{'_'.join(parts[1:-1])}.md"
                        new_asset_path = asset_dir / new_asset_name
                        
                        # 重複を避けるため、既に存在しない場合のみコピー
                        if not new_asset_path.exists():
                            import shutil
                            shutil.copy(draft_file, new_asset_path)
                            self.log(f"   ✨ スコア {score} のドラフトを資産化しました: {new_asset_name}")
                            self.syncer.ship_file(new_asset_path, category="Asset") # Notionへ同期
                            harvested_count += 1
                        else:
                            self.log(f"   ⏩ 既に資産化済みのドラフトをスキップしました: {new_asset_name}")
                        
                        # 資産化された元のドラフトファイルを削除するかどうかはポリシーによる。
                        # ここでは残しておくが、必要に応じて有効化できるオプションとしてコメントアウト。
                        # if os.getenv("DELETE_HARVESTED_DRAFTS", "false").lower() == "true":
                        #     draft_file.unlink() # 元ファイルを削除
                        #     self.log(f"   🗑️ 元のドラフトを削除しました: {draft_file.name}")

            except ValueError:
                self.log(f"⚠️ ドラフトファイル名からスコアを解析できませんでした: {draft_file.name}")
            except Exception as e:
                self.log(f"⚠️ ドラフト資産化中に予期せぬエラーが発生しました ({draft_file.name}): {e}")
        
        self.log(f"🧺 資産収穫プロセスが完了しました。新たに {harvested_count} 件の資産が追加されました。")

```

### 修正のポイント

1.  **メソッドの定義**: `OmniAgent` クラス内に `harvest_assets` メソッドを新規に定義します。
2.  **目的の実現**:
    *   `03_factory/daily_posts` ディレクトリから、自動生成されたドラフトファイルを読み込みます。
    *   ファイル名（例: `draft_YYYYMMDD_HHMM_sSCORE.md`）からスコアを抽出し、設定された閾値（`MIN_SCORE_FOR_ASSET`）以上のものを「高評価ドラフト」と判断します。
    *   高評価ドラフトは `01_spirit/assets` ディレクトリに、`evergreen_` プレフィックスを付けた新しいファイル名でコピーされます。これにより、永続的な知識資産として区別されます。
    *   `self.syncer.ship_file` を使用して、新しく資産化されたファイルを Notion に同期します。
3.  **堅牢性**:
    *   資産格納ディレクトリ (`01_spirit/assets`) が存在しない場合は自動的に作成します。
    *   ファイル名解析のエラーや、その他の予期せぬエラーが発生した場合でも、処理が中断しないように `try-except` ブロックで囲んでいます。
    *   既に資産化されているファイル（同じ内容の `evergreen_` ファイルが存在する場合）はスキップし、重複を避けます。
4.  **ログ出力**: 各ステップで詳細なログを出力し、AdminAn としての作業状況を可視化します。

この修正により、エラーが解消され、`OmniAgent` の自律経営サイクルにおける「資産収穫」のステップが適切に機能するようになります。ご確認の上、システムに適用してください。
