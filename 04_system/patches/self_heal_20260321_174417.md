## エラー分析と修正案

### エラー分析

提供されたエラーメッセージ `AttributeError: 'OmniAgent' object has no attribute 'harvest_assets'` は、`OmniAgent` クラスのインスタンスが `harvest_assets` というメソッドを持っていないことを示しています。

トレースバックを見ると、エラーは `d:\my_work\02_intelligence\intelligence\omni_agent.py` の `run_cycle` メソッド内の以下の行で発生しています。

```python
self.harvest_assets()
```

このエラーは、`run_cycle` メソッド内で `harvest_assets` メソッドを呼び出そうとしていますが、`OmniAgent` クラスの定義の中に `harvest_assets` メソッドが存在しないために発生しています。

ソースコード全体を確認しましたが、`harvest_assets` という名前のメソッドは定義されていません。

### 修正案

このエラーを修正するには、以下のいずれかの方法が考えられます。

#### 修正案1：`harvest_assets` メソッドを `OmniAgent` クラスに追加する

これは、`harvest_assets` メソッドが意図的に存在しないのではなく、単に実装漏れである場合に最も適切な対応です。

**手順:**

1.  `OmniAgent` クラス定義の中に、`harvest_assets` メソッドを追加します。
2.  `harvest_assets` メソッドの具体的な実装は、そのメソッドが本来どのような役割を果たすべきかに依存します。提供されたコードの文脈（`"🧺 [AdminAn] Step 9/12: 高評価ドラフトの資産化..."` というログメッセージ）から推測すると、高評価なドラフトなどを何らかの形で「資産化」する処理を行うことが想定されます。

**Pythonコード例:**

```python
# ... (省略) ...

class OmniAgent:
    # ... (既存の __init__ メソッドなど) ...

    # --- 追加するメソッド ---
    def harvest_assets(self):
        """
        高評価ドラフトや成果物を資産として収集・整理する
        （具体的な実装は、収集対象や保存方法によって異なります）
        """
        self.log("🧺 [AdminAn] 資産収穫プロセスを実行中...")
        
        # 例：評価の高いドラフトを特定のディレクトリにコピーする

        # 例として、draft_*.md ファイルからスコアが 90 以上のものを収集する処理を想定します。

        asset_dir = ROOT_DIR / "01_spirit" / "assets" # 資産を保存するディレクトリ（仮）

                    # 必要であれば、元のドラフトをアーカイブするなど
            except (ValueError, IndexError) as e:
                self.log(f"⚠️ ファイル名からスコアを抽出できませんでした（スキップ）: {draft_file.name} - {e}")
            except Exception as e:
                self.log(f"❌ 資産化中にエラーが発生しました ({draft_file.name}): {e}")

    # 9. AdminAn: 資産収穫 (Asset Harvesting)
    self.log("🧺 [AdminAn] Step 9/12: 高評価ドラフトの資産化...")
    self.harvest_assets() # この行が正常に実行されるようになります
    # ... (以降のメソッド) ...
```

*   上記の `harvest_assets` の実装は、あくまで推測に基づいた例です。実際の要件に合わせて、収集対象、保存場所、処理内容を具体的に定義してください。
*   `asset_dir` (`ROOT_DIR / "01_spirit" / "assets"`) のようなパスは、コード全体での命名規則や意図に合わせて調整してください。`01_spirit` ディレクトリは「基礎」のような意味合いで使われているようですが、資産の保存場所として適切か検討してください。

### どちらの修正案を選択すべきか？

*   **「高評価ドラフトの資産化」というログメッセージが存在する**ことから、この処理は本来意図されていた可能性が高いです。そのため、**修正案1** を採用し、`harvest_assets` メソッドを適切に実装することを強く推奨します。
*   もし、この処理が一時的に無効化されており、将来的に再度有効化される可能性がある場合は、`run_cycle` メソッドから削除するのではなく、`harvest_assets` メソッド自体をコメントアウトし、その理由をコメントで明記しておくと良いでしょう。

まずは、`harvest_assets` がどのような処理を行うべきか、開発チーム内で確認し、**修正案1** に沿って実装を進めるのが最も健全な対応となります。
