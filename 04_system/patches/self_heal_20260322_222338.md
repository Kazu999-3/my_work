# 🩹 自己修復プロポーザル

## エラー原因と修正案

### エラー原因

エラーメッセージ `AttributeError: 'OmniAgent' object has no attribute 'harvest_assets'` は、`OmniAgent` クラスのインスタンスが `harvest_assets` というメソッドを持っていないことを示しています。

トレースバックを見ると、`run_cycle` メソッドの以下の行でエラーが発生しています。

```python
self.harvest_assets()
```

これは、`run_cycle` メソッド内で `self.harvest_assets()` を呼び出そうとしていますが、`OmniAgent` クラスの定義には `harvest_assets` メソッドが存在しないため発生しています。

### 修正案

`OmniAgent` クラスに `harvest_assets` メソッドを追加します。

#### 修正コード (Python)

`OmniAgent` クラスの `run_cycle` メソッドの呼び出し箇所（9番目のログメッセージの直前）より前に、以下の `harvest_assets` メソッドを定義してください。

```python
    # ... (OmniAgent クラスの他のメソッド定義) ...

    def harvest_assets(self):
        """
        高評価ドラフトや成功事例を「資産」として収穫する
        （現在は仮実装。将来的に Notion などへの登録や、特定のファイルへの保存を行う）
        """
        self.log("🧺 [AdminAn] 資産収穫プロセスを実行中 (仮実装)...")
        # ここで、高評価のドラフトや成功したレポートなどを特定し、

        # 例:
```

**修正のポイント:**

1.  **メソッドの追加:** `OmniAgent` クラス内に `harvest_assets` という名前のメソッドを定義しました。
2.  **仮実装:** 現在、このメソッドの具体的な処理は定義されていません。コメントアウトされた部分に、資産として認識すべきドラフトやレポートを特定し、それを管理するための処理（例: 特定のディレクトリへのコピー）を実装する例を示しています。
3.  **ログ出力:** 処理の実行状況を把握できるよう、ログ出力が含まれています。

### 適用手順

1.  上記の `harvest_assets` メソッドのコードを、`OmniAgent` クラス内の適切な位置（例えば、`generate_daily_report` メソッドの後や、`assemble_products` メソッドの前など）にコピー＆ペーストしてください。
2.  ソースコードを保存します。
3.  再度システムを実行し、エラーが解消されていることを確認してください。
