## エラー分析と修正案

### エラー内容の分析

提供されたエラーメッセージ `AttributeError: 'OmniAgent' object has no attribute 'harvest_assets'` は、`OmniAgent` クラスのインスタンスが `harvest_assets` というメソッドを持っていないのに、そのメソッドを呼び出そうとしたことを示しています。

トレースバックを確認すると、エラーは `omni_agent.py` の `run_cycle` メソッドの145行目で発生しています。

```python
# ... (省略)
# 9. AdminAn: 資産収穫 (Asset Harvesting)
self.log("🧺 [AdminAn] Step 9/12: 高評価ドラフトの資産化...")
self.harvest_assets() # ここでエラーが発生
# ... (省略)
```

このコードでは、`OmniAgent` インスタンス (`self`) の `harvest_assets` メソッドを呼び出しています。しかし、`OmniAgent` クラスの定義を見ると、`__init__` メソッドでも、他のメソッドでも `harvest_assets` という名前のメソッドは定義されていません。

### 修正手順

1.  **修正案1（推奨）を選択する場合:**
    *   提供したPythonコードをコピーし、`OmniAgent` クラス定義の適切な位置（例えば、`calculate_dynamic_wait` メソッドの後など）に `harvest_assets` メソッドを追加してください。

### 補足

*   `harvest_assets` メソッドの正確な実装内容は、このコードスニペットだけでは判断できません。ファイルシステム操作、データベースへの記録、あるいは他のサービスとの連携などが考えられます。
*   もし、`harvest_assets` が別のファイルで定義されているクラスのメソッドである場合、そのクラスを `OmniAgent` の `__init__` メソッドでインスタンス化しているか、あるいは `OmniAgent` クラス内でそのメソッドを直接呼び出すのではなく、インスタンス経由で呼び出す必要があります。しかし、今回のエラーメッセージからは、`OmniAgent` クラス自体のインスタンスに `harvest_assets` 属性がないことが原因と断定できます。

まずは、`harvest_assets` メソッドを `OmniAgent` クラスに追加（修正案1）するのが、最も直接的で推奨される解決策です。
