# 🩹 自己修復プロポーザル

## エラー分析と修正案

### エラーの原因

エラーメッセージ「'OmniAgent' object has no attribute 'harvest_assets'」とトレースバックから、`OmniAgent` クラスのインスタンスにおいて `harvest_assets` というメソッドが存在しないにも関わらず、`run_cycle` メソッド内でそのメソッドを呼び出そうとしていることが原因であると特定できます。

具体的には、`run_cycle` メソッドの以下の行がエラーを引き起こしています。

```python
            # 9. AdminAn: 資産収穫 (Asset Harvesting)
            self.log("🧺 [AdminAn] Step 9/12: 高評価ドラフトの資産化...")
            self.harvest_assets() # <--- ここでエラーが発生
```

### 修正案

このエラーを修正するには、`harvest_assets` メソッドを `OmniAgent` クラス内に定義する必要があります。

#### 修正コード

以下のコードを、`OmniAgent` クラスの `async def admin_an_auto_evolve_project(self):` メソッドの直前（または適切な位置）に挿入してください。

```python
    def harvest_assets(self):
        """
        AdminAn: 高評価ドラフトや成功事例を資産として収穫・管理する
        """
        self.log("🧺 [AdminAn] 資産収穫プロセスを開始します...")
        
        # 例: 03_factory/daily_posts ディレクトリにある、スコアが高いドラフトをアーカイブする
        draft_dir = ROOT_DIR / "03_factory" / "daily_posts"
        archive_dir = ROOT_DIR / "03_factory" / "archives"
        archive_dir.mkdir(parents=True, exist_ok=True)
        
        for draft_file in draft_dir.glob("draft_*_s*.md"):
            try:
                score_match = re.search(r"_s(\d+)\.md$", draft_file.name)
                if score_match:
                    score = int(score_match.group(1))
                    # スコアが90以上のドラフトを資産として収穫（アーカイブ）する例
                    if score >= 90:
                        archive_path = archive_dir / f"harvested_{draft_file.name}"
                        draft_file.rename(archive_path)
                        self.log(f"✨ 高評価ドラフトを資産化しました: {archive_path.name} (Score: {score})")
                        # 収穫した資産をNotionに同期する（必要に応じて OmniSyncPro を使用）
                        # self.syncer.ship_file(archive_path, category="Harvested Asset")
            except Exception as e:
                self.log(f"⚠️ ドラフト '{draft_file.name}' の収穫中にエラー: {e}")
        
        # 将来的に、成功したレポートやその他価値のあるファイルをここに集約する
        # 例:
        # report_dir = ROOT_DIR / "03_factory" / "reports"
        # for report_file in report_dir.glob("trend_report_*.md"):
        #     # 何らかの基準で「資産」と判断されたら移動・コピー
        #     pass
            
        self.log("🧺 [AdminAn] 資産収穫プロセスを完了しました。")
```

**解説:**

1.  **メソッドの追加:** `harvest_assets` という名前で新しいメソッドを `OmniAgent` クラスに追加しました。
2.  **ログ出力:** 処理の開始と完了をログに出力するようにしました。
3.  **資産収穫のロジック (例):**
    *   `ROOT_DIR / "03_factory" / "daily_posts"` ディレクトリから、スコアが付与されたドラフトファイル（`draft_*_s*.md`）を検索します。
    *   ファイル名からスコアを抽出し、もしスコアが90以上であれば、そのファイルを `ROOT_DIR / "03_factory" / "archives"` ディレクトリに移動（リネーム）し、資産として扱います。
    *   `re` モジュールを使用するため、ファイルの先頭に `import re` を追加してください。
4.  **拡張性:** このメソッドはあくまで一例です。将来的に、成功したレポート、特定の条件を満たすドキュメントなどを「資産」として管理・収穫するためのロジックをここに追加していくことができます。

#### 修正手順

1.  提供された `OmniAgent` クラスのソースコードを開きます。
2.  `async def admin_an_auto_evolve_project(self):` メソッドの定義の直前（または、他の `self.` メソッド定義の近く）に、上記の `harvest_assets` メソッドのコードをコピー＆ペーストします。
3.  ファイルの先頭に `import re` を追加します（もし既に追加されていなければ）。
4.  ファイルを保存します。

この修正により、`run_cycle` メソッド内で `self.harvest_assets()` が正しく呼び出され、`AttributeError` が解消されるはずです。
