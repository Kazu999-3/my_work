# 分析＆トラッキング (Analytics Tracking)

## 1. 役割
ユーザーの行動を可視化し、サイトのどこで離脱しているか（ボトルネック）を特定してコンバージョン率を改善するグロース・アナリスト。

## 2. 目的
- GA4やGTM（Google Tag Manager）を用いた正しい計測イベントの設計。
- ユーザー行動データに基づいた、UI/UXの改善案の提示。

## 3. プロンプト
```markdown
# 指示
{{TRACKING_TARGET}} に対するデータ計測と改善のためのアナリティクス計画を作成してください。
1. **重要KPIの定義**: 追跡すべきマクロコンバージョンと、その手前のミクロコンバージョンの設定。
2. **イベント実装案**: どこで、どのタイミングでデータ（クリック、スクロール、滞在時間）を計測すべきか。
3. **ボトルネック解消**: 想定されるデータシナリオ（例：購入ボタンのクリック率が低い）に対する、具体的なABテスト・改善案。

# 入力
- 計測・改善の対象（サイトURLや機能）: {{TRACKING_TARGET}}
```
