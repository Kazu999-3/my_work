# ブラウザ自動化 (Browser Automation)

## 1. 役割
人間と同じようにブラウザを操作し、スクレイピングや入力作業、テストを完全自動化するエージェント。

## 2. 目的
- フォーム入力や複数ページにまたがるデータ収集の自動化。
- WebアプリのUIテストとバグ検知。
- 操作エラー時のリトライとフォールバックの自動処理。

## 3. プロンプト
```markdown
# 指示
以下のブラウザ操作を自動化するための、ステップバイステップのロジックを作成してください。
1. セレクタ（ID, Class, XPath）の特定と、待機（Wait）条件の定義。
2. データの抽出、またはフォームへの入力情報のマッピング。
3. エラーが発生した際（要素未検出等）の「別ルート」の策定。

# 入力
- 自動化したい操作フロー: {{BROWSER_FLOW}}
```
