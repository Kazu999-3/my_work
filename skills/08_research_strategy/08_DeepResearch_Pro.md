# 🔍 究極のディープリサーチPro：Ultimate Deep Research Pro (08_DeepResearch_Pro.md)

このスキルは、情報の表面的な要約を超え、「プレイ改善」と「知見の資産化（note/記事用）」のために、インターネット上の「幅広く深い」情報を多角的に収集・構造化するためのプロ仕様リサーチエンジンです。

## 1. コア・ミッション
- **情報の非対称性の解消**: 英語圏（Reddit, Dev Blogs）やSNS（X, Discord）の深層にある、日本語圏では得られない最新情報を掘り起こす。
- **データ透明性の確保**: 全ての事実に対して出典URLを付記し、LoL関連なら必ずパッチバージョンを特定または明記する。
- **実行可能な洞察の提供**: 単なる「知識」ではなく、「次からどう動けば勝てるか」という具体的なテクニック（プレイ改善）と、読者が価値を感じる独自の考察（note用）を分離して提供する。

## 2. リサーチ・プロトコル

### ステップ1：ソースの多角化（取捨選択せず、幅広く集める）
以下の優先順位で情報をスキャンする：
1. **海外掲示板/コミュニティ**: Reddit (r/leagueoflegends, r/[Champion]mains), Wiki, Patch Notes.
2. **SNS/リアルタイムトレンド**: X (以前のTwitter), YouTube (VOD/高ランク解説), プロ選手の配信クリップ。
3. **統計・データ基盤**: LoLalytics, OP.GG, Riot Developer Blog.

### ステップ2：構造化出力（LoL特化）
全ての情報は以下の項目を含めて整理する：
- **[Patch]**: 情報がどのパッチに基づいているか（16.5等）。分からない場合は「不明」と明記。
- **[Source]**: 出典URL。複数の場合は全て列挙。
- **[Play Improvement]**: プレイの質を即座に上げるテクニック、隠れた仕様、バグ、マッチアップの微細な有利不利。
- **[Note Insight]**: note記事の「目玉」になる、数字に基づいた独自の考察や、意外なトレンド要因。

# 指示

調査対象: {{RESEARCH_TARGET}}

# 実行ルール
1. **URLの絶対保持**: 全ての重要な情報には必ず出典元URLを併記すること。
2. **LoLパッチの特定**: 対象がLoL関連の場合、必ず「その情報のパッチバージョン」を特定して記載すること（例：Patch 16.5）。特定できない場合は「パッチ不明」とすること。
3. **幅広く深く**: 情報の取捨選択は最後の段階で行う。まずは関連する可能性のある「海外のテクニカルな情報」や「SNSの意外な反応」を漏らさずピックアップすること。
4. **二元論的出力**: 「プレイ改善（具体的な操作や知識）」と「note用洞察（価値ある分析）」に分類して出力すること。

# 実行プロトタイプ（網羅性強化版）

1. **【総合診断：現在の立ち位置】**
   - [Patch 〇.〇] ティア / 勝率 / 使用率 / 前パッチからの変更点

   - **A. ビルド・アイテムの変遷**: (出典URL)
   - **B. マッチアップ（天敵・カモ）**: (出典URL)
   - **C. ルート・ジャングル技術**: (出典URL)
   - **D. 海外（Reddit/英Wiki）の深層情報**: (出典URL)
