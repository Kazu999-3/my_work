# 記事レビュー & 弱点分析スキル

このスキルは、note等の記事を「読ませる・動かす」高品質なコンテンツへと進化させるための、5つの黄金観点に基づいたレビューを自動化するものです。

## 1. 評価観点とチェック項目

### ① 冒頭3行で読み手のスクロールが止まるか
- [ ] 最初の3行以内に「解決策」や「ベネフィット」が提示されているか。
- [ ] インパクトのある導入になっているか。

### ② 読み手が「自分のことだ」と感じる場面描写があるか
- [ ] 読者の悩みやあるある（シチュエーション）が想起できる描写があるか。
- [ ] 「共感」から記事に入り込めるか。

### ③ 主張に具体性（数字・固有名・実例）があるか
- [ ] 抽象的な「便利」「凄い」ではなく、具体的な数値（10倍、12個など）や名称があるか。
- [ ] 実際に何が起きたかのエピソードが含まれているか。

### ④ 論理の流れに飛躍や矛盾がないか
- [ ] 前後の文脈がスムーズに繋がっているか。
- [ ] 「なぜそれが必要なのか」のWhyが明確か。

### ⑤ 最後に読み手の行動を促す導線があるか
- [ ] 読み終わった後、読者に何を求めているか明確か（スキ・フォロー・購入など）。
- [ ] 行動したくなる「理由」が添えられているか。

### ⑥ 【追加】タイトルのSEO・クリック率（CTR）対策
- [ ] 検索されやすいキーワード（AI, Notion, Discord, 効率化など）が含まれているか。
- [ ] ターゲットの「悩み」や「理想の状態」を刺激するワードがあるか。
- [ ] 読み手がクリックしたくなる「数字」や「実績」が効果的に使われているか。

## 2. 出力フォーマット
各観点について、以下の形式で出力してください。

- **現状の問題**: 具体的にどこの記述がどう不十分か。
- **改善の方向性**: どのように書き換えるべきか（具体的な例文案など）。
- ※問題がない場合は「問題なし」と明記。

## 3. 分析後の改善フロー（必須実行）
分析を完了した後は、必ず以下のステップを実行してください。

### ステップ2：改善版（vx）の出力
分析結果をすべて反映した改善版を `vx`（例：v4）として出力してください。

- **制約事項**: 
  - 元の記事の意図・トーン・構成順序を厳格に維持すること。
  - 分析に基づかない勝手な要素追加、表現の大幅な変更、削除は禁止。
  - 末尾に「変更点」として、具体的にどこをどう変えたか、どこの分析結果に基づいているかをまとめて記述すること。

## 4. 使用方法
「この記事を記事レビュースキルで分析・改善して」と指示してください。
