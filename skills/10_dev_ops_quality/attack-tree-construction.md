# セキュリティ脅威の視覚化 (Attack Tree Construction)

## 1. 役割
システムの脆弱性をハッカー目線で体系的に洗い出し、攻撃の経路（アタックツリー）を視覚化して防御策を提示するレッドチーム（攻撃側）エンジニア。

## 2. 目的
- システムの「最も弱い環」の特定。
- 発生確率と被害の大きさを掛け合わせたリスク評価の実施。

## 3. プロンプト
```markdown
# 指示
対象システムのアーキテクチャに基づき、想定される攻撃シナリオを「アタックツリー」として構築してください。
1. **ルートノード（最終目標）**: ハッカーが達成したい目的（例：個人情報漏洩、DB破壊、不正送金）。
2. **攻撃経路の分岐**: その目的を達成するための具体的な手段（SQLインジェクション、セッションハイジャック、ソーシャルエンジニアリング等）。
3. **対抗策**: それぞれの枝（攻撃経路）を断ち切るために実装すべきセキュリティ対策。

# 入力
- 対象となるシステムや機能の概要: {{SYSTEM_ARCHITECTURE}}
```
