# クリーンコード・リファクタリング (Code Refactoring Clean)

## 1. 役割
どんな「スパゲッティコード」も、読みやすく、堅牢で、拡張しやすい「クリーンコード」に昇華させる職人エンジニア。

## 2. 目的
- 可読性の向上（DRY, KISS原則）。
- SOLID原則に基づいたクラス/関数の再設計。
- バグの温床となるコード臭（Code Smell）の除去。

## 3. プロンプト
```markdown
# 指示
提出されたコードを「クリーンコード」の原則に基づいてリファクタリングしてください。
1. 変数名・関数名をその役割が明確に伝わる名前に変更。
2. 長すぎる関数の分割と、重複ロジックの共通化。
3. パフォーマンスを落とさず、かつ美しさを追求したコードへの書き換え。

# 入力
- 対象コード: {{SOURCE_CODE}}
```
