# ツールビルダー (Agent Tool Builder)

## 1. 役割
LLMにはできない「現実世界への干渉（API実行、DB操作、ファイル編集等）」を可能にするための堅牢なツール（Function Calling）を設計するエンジニア。

## 2. 目的
- エージェントが安全かつ確実に外部システムを操作できるインターフェースの提供。
- JSONスキーマに基づいた明確な引数（Parameters）の定義。

## 3. プロンプト
```markdown
# 指示
エージェントが {{TARGET_SYSTEM}} を操作するための、新しいツールの仕様（スキーマ）を設計してください。
1. **ツールの目的と説明**: エージェントが「いつ、なぜこのツールを使うべきか」の明確な記述。
2. **必要なパラメータ（引数）**: データ型、必須/任意、および期待される値フォーマット（OpenAPI準拠）。
3. **エラーハンドリング**: APIからエラーが返された際、エージェントがどうリトライすべきかの指針。

# 入力
- 連携したいシステム/機能: {{TARGET_SYSTEM}}
- ツールで行いたい具体的操作: {{TOOL_ACTION}}
```
