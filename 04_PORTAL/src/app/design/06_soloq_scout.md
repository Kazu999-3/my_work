# 🔍 ソロQ対面リアルタイム偵察 (Live Match Lookup)

管理者が自身の Riot ID（または他のプレイヤーのID）を入力して実行し、現在進行中のライブゲームを検知して、敵のジャングラーのプレイスタイル、開始バフ予測、初動Gank（レーン関与）予測、および具体的な対策Tipsを出力する機能です。

---

## 1. 概要とデータ連携

*   **データソース（取得元）**:
    *   **Riot Games 公式 API**:
        *   Riot ID 解決: `/riot/account/v1/accounts/by-riot-id/`
        *   進行中ゲーム (Spectator-V5): `/lol/spectator/v5/active-games/by-puuid/`
        *   敵ジャングラーの過去戦績: `/lol/match/v5/matches/by-puuid/` および `/timeline`
*   **データ追加・更新トリガー**:
    *   **手動リアルタイムフェッチ（使い捨てデータ）**:
        *   管理者がサモナー名（Riot ID）を入力し、「偵察開始」ボタンを **手動で押したタイミングでのみ** リアルタイムに Riot API からデータを引き出し、その場でメモリ上だけで計算・表示します。
        *   データベース（Supabase）には分析結果や検索履歴などは一切保存されず、完全に「その場限り」の使い捨てデータです（自動巡回や定期自動保存はされません）。
        *   ※ `RIOT_API_KEY`（環境変数）が未設定の開発・デモ環境では、Lee Sin を対面とするダミーの模擬ライブデータを自動生成して返却する「モックフォールバック機能」が自動で起動します。
*   **公開区分**: **管理者専用 🔑** (パス: `/admin/soloq`)

---

## 2. 敵ジャングラー特定 ＆ 戦術予測アルゴリズム

### 2-1. 敵ジャングラーの特定
1.  API（`/lol/spectator/v5/active-games/by-puuid/`）から、進行中のゲームの全参加者（10名）のサモナースペルおよびチームIDを取得します。
2.  検索されたサモナーのチームIDを特定し、敵チーム（もう一方のチームID）にいるプレイヤーの中から、召喚士スペルに **Smite（ID: 11）を装備しているプレイヤー** を「敵ジャングラー」として自動特定します。

### 2-2. 開始バフ（赤/青）＆ 初動Gank（レーン関与）の予測
敵の「使用チャンピオン」と「過去のプレイスタイルタグ」から、AI/ロジックで初動ルートを推測します。

*   **戦闘狂タイプ (例: Lee Sin, Nidalee, Shaco 等の早期Gankチャンピオン)**:
    *   **予測バフ**: 「青バフ（バフ3キャンプ速攻）スタート予測」または「ボット側リーシュあり赤バフスタート」。
    *   **予測Gank**: 「トップまたはミッドへのLV3早期Gank」「リバーでの遭遇戦」。
*   **ファーム・成長優先タイプ (例: Graves, Karthus 等のフルクリアチャンピオン)**:
    *   **予測バフ**: 「赤バフ（フルクリア周回）スタート予測」。
    *   **予測Gank**: 「フルクリア後のLV4でのスカトル争い、または最初のオブジェクト関与」。
*   **対策アドバイスの生成**:
    敵のプレイスタイルタグ（例: `early-brawler` [戦闘狂] や `speed-demon` [ファーマー]）に基づき、開始3分前後にどのレーンにワードを置くべきかなどの具体的な戦術テキストを動的に生成します。
